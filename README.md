# MIT Cycling Club Website

Hugo static site for [cycling.mit.edu](https://cycling.mit.edu). Built with Hugo v0.163+ extended.

## Quick start

```bash
# Fetch latest race results and build
./build.sh --minify

# Local dev server
hugo server
```

## Configuration

Key settings in `hugo.toml`:

| Param | Purpose |
|-------|---------|
| `heroImage` | Hero photo path (relative to `assets/`), e.g. `img/hero-team.jpg` |
| `email` | General club contact email |
| `officersEmail` | Officers list shown on the Contact page |
| `sponsorshipEmail` | Sponsorship inquiries email |
| `roadResultsTeamId` | road-results.com team ID for widget embed |
| `crossResultsTeamId` | crossresults.com team ID for widget embed |

## Content

| Directory | What goes here |
|-----------|---------------|
| `content/blog/` | Blog posts (Markdown) |
| `data/sponsors.yaml` | Sponsor list with logos and URLs |
| `data/roster.yaml` | Team roster by season |
| `data/routes.yaml` | Strava route embeds |
| `data/faq.yaml` | FAQ entries |
| `data/race_results_live.json` | Auto-generated by `scripts/fetch_results.py` |

## Images

All images that need correct URL prefixing live in `assets/img/`:

- `assets/img/hero-team.jpg` — hero photo (swap via `heroImage` param)
- `assets/img/blog/` — blog post header images
- `assets/img/blog/inline/` — inline images embedded in blog posts
- `assets/img/blog/posts/<slug>/` — per-post inline images (where downloaded)
- `assets/img/sponsors/` — sponsor logos

Images in `assets/` are served through Hugo's resource pipeline, which correctly prepends the site base path. Do not put images that need URL-prefixing in `static/`.

## Race results

Results are scraped from [results.bikereg.com/team/40588](https://results.bikereg.com/team/40588) via Playwright:

```bash
python3 scripts/fetch_results.py
```

The GitHub Actions workflow (`.github/workflows/fetch-results.yml`) runs this nightly and commits the updated JSON.

## Deployment

Push to `main` — the `.github/workflows/deploy.yml` workflow fetches results, builds with Hugo, and deploys to GitHub Pages automatically.
